# Audit IA & Game Engine — 2026-05-19

> État des lieux des deux moteurs de Nuffle Arena (`game-engine`,
> `sim-engine`) et de la pertinence de l'arbre de décision IA.
> Référence durable pour les sprints à venir : prioriser le levier
> "skill awareness" sur le block proba avant tout autre chantier IA.

## Périmètre

- `packages/game-engine/` — moteur BB online (multijoueur, board complet)
- `packages/sim-engine/` — simulateur Pro League offline (batch)
- Focus : architecture moteur + arbre de décision IA + pertinence
  compétitive

## TL;DR

- **Moteur de jeu** : solide, complet, testé, immuable — aucune urgence.
- **IA game-engine** : opérationnelle mais naïve (1-ply greedy + 7
  composantes sans skill awareness). Battra un débutant, perdra contre
  un joueur intermédiaire. **Levier #1 = skill awareness sur block
  proba** (gain massif pour ~200 lignes).
- **IA sim-engine** : architecturalement plus riche (3-pass behavior
  tree, 15 paramètres tactiques par équipe, race profiles) — mais opère
  sur abstraction yards, pas board réel.
- **Action prioritaire** : converger les deux IA + ajouter skill
  awareness à `estimateBlockKnockdown`. ~1-2 sprints, impact compétitif
  net.

## 2. Moteur de jeu — robuste

### Forces

- **Immutabilité stricte** — `applyMove(state, move, rng)` → nouvel
  état (tests dédiés `mutations-immutability.test.ts`,
  `blitz-handler-immutability.test.ts`).
- **41 mécaniques BB3 S3** testées individuellement (Dauntless, Frenzy,
  Wrestle, Shadowing, Stand Firm, Fend, Break Tackle, Juggernaut…).
- **31+ types de coups** clairement séparés en handlers modulaires
  (`packages/game-engine/src/core/types.ts:238-271`).
- **Event replay** compressé (`core/replay.ts`).
- **Cycle pre-match** complet : weather, journeymen, inducements,
  prayers, kicking-team, setup, kickoff.
- **Couverture règles** : aucun gap visible BB2020/BB3 S3.

### Faiblesses

- Zéro FIXME/HACK — structurellement propre.
- 2 TODOs mineurs (`rosters/priority-teams.ts`, `utils/logging.ts`).
- Pas de tests d'intégration "IA joue un match complet" côté
  game-engine seul.

## 3. Arbre de décision IA — game-engine

### Architecture

```
src/ai/
├── evaluator.ts   — score d'un état (7 composantes pondérées)
├── lookahead.ts   — minimax 2-ply uniquement sur END_TURN
├── difficulty.ts  — easy/medium/hard via noise + blunder + timidity
└── index.ts       — orchestration pickBestMove / pickBestMove2Ply
```

### Heuristique d'évaluation (`evaluator.ts:35-77`)

7 composantes pondérées (poids "magiques") :

| Composante | Poids clé | Remarque |
|---|---|---|
| `score` | TOUCHDOWN = 1000 | Différence TD × poids |
| `possession` | POSSESSION = 300 | Bonus binaire |
| `ballProgress` | 15 / case | Distance porteur → endzone |
| `playerCount` | actif +30 / stunned -15 / KO -40 | Effectif sur terrain |
| `carrierSafety` | ally +20 / TZ adverse -35 | Adjacences porteur |
| `attrition` | casualty -150 / sent_off -120 | Pénalité cumulée |
| `positioning` | 2 / pas | Approche porteur ou progression |

### Scoring de coups (`evaluator.ts:363-525`)

- **BLOCK** : knockdown × valeur cible + selfRisk
- **Knockdown proba** (`L356-360`) hardcodée : `2D = 0.7`, `1D win = 0.55`,
  `1D égal = 0.33`, `2D lose = 0.08` — **ne tient compte d'AUCUN skill**
  (Block, Wrestle, Dauntless, Frenzy, Stand Firm…).
- **BLITZ** : moveDelta + blockDelta
- **PASS/HANDOFF** : delta position − `PASS_RISK_PENALTY` (8) →
  souvent dominé par 300 de possession ⇒ pass quasi jamais tentée.
- **FOUL** : 90 carrier / 30 non-carrier − 40 turnover risk
- **LEAP** : moveDelta − 15 (pénalité arbitraire)
- Default action inconnue : `-50`

### Profondeur

- **1-ply greedy** par défaut (`L528-545`)
- **2-ply asymétrique** : seul END_TURN simule la meilleure réponse
  adverse (`lookahead.ts:143-162`)
- Choix rationnel (END_TURN = pivot du tour, déterministe) mais
  **aveugle aux pièges offensifs** type "double-tackle bait sur le
  porteur"

### Difficulté (`difficulty.ts`)

| Niveau | noise | blunder | timidity | endTurnBias |
|---|---|---|---|---|
| easy | 80 | 0.35 | 0.35 | 40 |
| medium | 20 | 0.10 | 0.05 | 0 |
| hard | 0 | 0 | 0 | 0 |

### Verdict pertinence

**Suffisante pour "practice mode" / IA en ligne basique. Inacceptable
pour battre un humain compétent.**

Faiblesses critiques :
1. **Zéro skill awareness** → même comportement face à Skavens (AG4)
   et Dwarves (Tackle partout).
2. **Knockdown proba mensongère** → IA sur-estime ses 1D en bashing,
   sous-estime Wrestle/Block stacks.
3. **Pas de GFI/dodge probability modeling** → ne sait pas évaluer un
   chain dodge ou un blitz à 2 GFI.
4. **Pas de reroll budgeting** → joue identiquement avec 4 RR
   restantes ou 0.
5. **PASS pénalisée à mort** → mode bash-only de fait.
6. **Aucun lookahead sur les pièges** → 1-ply greedy se fait punir.
7. **Heuristique LEAP/positioning** → constants magiques non validés.

## 4. IA sim-engine — plus mature, abstraction yards

### Architecture 3-pass (`packages/sim-engine/src/ai/play.ts:42-157`)

```
Pass 1: evaluateSituation  → flags (possession, redZone, lateInHalf…)
Pass 2: strategy.score()   → 6 stratégies via softmax (T = riskAppetite/40)
Pass 3: pattern + moment   → softmax sur block/dodge/pass/foul/gfi/pickup
```

6 stratégies (`strategy/strategies.ts`) : `cage-build`, `breakaway`,
`defensive-screen`, `blitz-train`, `stall`, `foul-fest`.

### Layer tactique (`packages/sim-engine/src/tactics/`)

| Module | Rôle | Statut |
|---|---|---|
| `tactical-profile.ts` | 15 params [0,100] par équipe | ✅ Zod-validé |
| `ai-weights.ts` | Profile → `EvalWeights` game-engine | ✅ ×0.4 → ×1.6 |
| `race-profiles.ts` | 16 races avec template tactique | ✅ |
| `opening-book.ts` | Bonus race-spécifique tours 1-3 | ✅ |
| `momentum.ts` | Hot/cold par joueur | ✅ |
| `temperature.ts` | Softmax stable | ✅ |
| `underdog.ts` | Retry silencieux 3% si TV gap > 200 | ⚠️ Tuning instable |
| `player-form.ts` | Form [1..5] par joueur | ❌ Présent, non intégré |

### Deux drivers

- **Hybrid** (`driver/hybrid-driver.ts:918-1019`) : abstraction yards,
  16 tours fixes, 1-2 "key moments" probabilistes/tour.
- **Full** (`driver/full-driver.ts:261-540`) : appelle game-engine + son
  IA, garde-fous `MAX_ACTIONS_PER_MATCH=1500`, stale-detection 4
  END_TURN.

### Tests sim-engine

- **Bench** : baseline ±5%, FUMBBL reference (Dwarf archetype)
- **Compare drivers** : delta hybrid vs full, outcome flip %
- **Perf** : 500 matches < 2s

## 5. Duplication & cohérence inter-engines

⚠️ **Resolvers sim-engine ≠ mechanics game-engine** — code dupliqué non
partagé :

- `sim-engine/src/resolvers/{block,dodge,pass,pickup,gfi,foul}.ts`
  (~1800 LOC)
- `game-engine/src/mechanics/...` (41 fichiers)
- **Risque de divergence** : sémantique armor roll, casualty outcomes,
  apothecary.
- Hybrid driver utilise resolvers sim-engine ; full driver utilise
  game-engine `applyMove` ⇒ deux vérités sur les mêmes règles BB.

## 6. Recommandations priorisées

### Quick wins (haute valeur, coût faible)

1. **Test d'intégration "IA vs IA full match"** dans game-engine —
   détecte tout comportement aberrant en 1 commit.
2. **Skill-aware `estimateBlockKnockdown`** (`evaluator.ts:356-360`) —
   injecter `hasSkill(attacker, 'Block')`, `hasSkill(defender,
   'Wrestle'|'Dodge')`, Dauntless avant le lookup → impact massif sur
   qualité IA.
3. **Probability modeling GFI/dodge** dans `scoreMove` — calculer
   succès × valeur attendue plutôt que delta brut.
4. **Reroll budget** dans heuristique — éviter de cramer le dernier RR
   sur action faible.

### Refacto structurant

5. **Partager les resolvers** entre sim-engine et game-engine (extract
   `@bb/rules-core`) — éliminer divergence à terme.
6. **Importer le layer tactique du sim-engine dans game-engine
   online** — `tactical-profile` × `ai-weights` × `race-profiles`
   existent déjà et sont testés.
7. **Lookahead profond limité** : 2-ply sur top-K (5-10) candidats au
   lieu de END_TURN seul, avec cap perf.

### Long terme

8. **MCTS léger** pour les positions critiques (porteur dans la zone
   rouge, dernier tour) — branchement explosion BB acceptable avec
   rollout policy = heuristique actuelle.
9. **Apprentissage des poids** via self-play sur le bench du sim-engine
   — déjà 80% de l'infra est là (compare/, bench/, FUMBBL ref).
