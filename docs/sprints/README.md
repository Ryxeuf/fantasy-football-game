## Sprint 1 — DONE

Fixed 5 CRITICAL + 7 HIGH bugs across 7 files:

### Game Engine
- `performArmorRollWithNotification` used 1D6 instead of 2D6 (every armor roll wrong)
- `rollBlockDiceManyWithRolls` + `performBlockRoll` consumed 2 RNG calls/die (corrupted seed)
- `handleBlock` extra RNG consumption (same issue, different location)
- `applyRollFailure` never called `performInjuryRoll` (broken armor = no KO/casualty)
- `handleBothDown` + Wrestle didn't set turnover on ball loss
- `handlePushBack` auto-moved attacker instead of offering follow-up choice

### Frontend
- Duplicate WebSocket connections (2 sockets per game)
- `normalizeState` mutated server payload in-place
- `GameScoreboard` hooks after conditional return (React crash)
- Drag-and-drop x/y axes swapped
- `InducementsPhaseUI` stale closure in useEffect
- `handleEndTurn` useMemo missing dependencies

### UI
- Tailwind dynamic classes purged in production builds

## Priority Order

1. **Sprint 2** — Game engine correctness is the foundation. Fixing state mutation and rule bugs prevents cascading issues.
2. **Sprint 3** — Frontend stability directly affects user experience. Memory leaks and race conditions cause visible bugs.
3. **Sprint 4** — Security hardening before production deployment. Required for any public-facing release.
4. **Sprint 5** — Test infrastructure enables safe future development. Lower urgency but high long-term value.
5. **Sprints 6 + 7** — New feature (badges/titres). Peut être implémentée en parallèle des sprints de stabilisation. Une seule PR combinant P1+P2.
